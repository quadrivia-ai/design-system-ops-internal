# Visual Report

## Context

Renders findings as a single self-contained HTML file that opens offline and passes strict
CSP. It visualises **labels and counts** — never invents numeric scores. Default output is
**pure HTML + CSS + inline SVG with no JavaScript**, which is the most CSP-safe and reliably
offline form.

## Configuration

Reads (all optional): the findings to render (from `output.report_dir` / `session-memory`);
`output.report_dir` (where the HTML is written).

## Auto-pull integrations

None — and by design it makes **no outbound calls**: no CDN, no fonts-from-the-web, no
analytics. Everything is inlined.

## Step 0 — Take findings, not raw source

Operate on already-produced findings (dimension health labels, finding severities, counts). If
none are provided, point to the audit/validate skills first; don't re-run analysis here.

## Step 1 — Compose three views (label/count based)

- **Health radar** — each dimension plotted by its health label (🟢 outer → 🔴 centre). It's a
  shape of labels, not a score.
- **Severity distribution** — counts of findings per severity (counts are measurements).
- **Priority matrix** — findings placed by severity × effort to show what to do first.

## Step 2 — Emit self-contained HTML

Hand-rolled inline SVG + CSS, zero JS by default. No external references of any kind. If
interactivity is explicitly requested, **vendor** the JS into the file (never a CDN link) and
note the CSP hash it requires.

## Produce the file

Write `<report_dir>/design-system-report.html` containing:

```html
<!doctype html>
<!-- GENERATED by design-system-ops · self-contained · no external calls -->
<meta http-equiv="Content-Security-Policy" content="default-src 'none'; style-src 'unsafe-inline'; img-src data:">
<style>/* all styles inlined */</style>
<!-- inline <svg> radar, bars, matrix; labels + counts only -->
```

Then a short summary:

```
## Visual Report — written to <path>

Rendered: <N dimensions, M findings>. Opens offline; no external calls.

### Scope
- Source findings: <which reports>
- Not done: no analysis re-run; no numeric scores invented
- Assumed: zero-JS static render unless interactivity was requested
```

## Closing note

Open the file with the network disabled to confirm it's truly self-contained — that's the
test that matters. The dashboard shows labels and counts; if a viewer wants a "score", that's
exactly what output-discipline forbids.

## Quality checks

- [ ] Single file; no CDN/external references; opens offline.
- [ ] Visuals use health labels + counts — no numeric scores.
- [ ] CSP-safe (zero-JS default; vendored + hash-noted if JS was requested).
- [ ] Output-discipline self-check passed.
