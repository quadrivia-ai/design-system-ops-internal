# Codemod Generator

> **Proposal only.** Emits code to review; runs nothing. Applying the codemod is a human
> action, dry-run-first (see
> [human-oversight-framework](../../knowledge-notes/human-oversight-framework.md)).

## Context

Generates the transform scripts and a `migrate.js` runner for a mechanical migration. The
output is code the team reviews and runs themselves — it never executes here, and the runner
defaults to dry-run so the first invocation changes nothing.

## Configuration

Reads (all optional): the migration spec (old → new); `source_globs` / `ignore_globs`;
`stack.token_files`, `stack.component_roots`.

## Auto-pull integrations

None.

## Step 0 — Pin the transform

State exactly what changes: `old → new` (token name, component import + prop mapping, banned
alias → token). Note cases it can't safely auto-transform (dynamic/computed usage) — those stay
manual and must be listed, not silently skipped.

## Step 1 — Generate the codemod

Emit an AST-based transform (preferred over regex for `.tsx`). Idempotent; leaves unmatched
code untouched; logs every change site.

## Step 2 — Generate the runner (dry-run default)

```js
// migrate.js — GENERATED by design-system-ops · REVIEW BEFORE RUNNING
// Default is DRY-RUN: prints changes, writes nothing.
// Apply only after reviewing the dry-run diff:  node migrate.js --apply
const APPLY = process.argv.includes("--apply");
// ...glob source files, run the transform, collect diffs...
if (!APPLY) {
  console.log(`DRY-RUN: ${changes.length} file(s) would change. Re-run with --apply to write.`);
  process.exit(0);
}
console.warn(`APPLYING to ${changes.length} file(s). Ensure a clean git tree first.`);
// ...write files...
```

## Step 3 — State scope and limits

How many files match, which cases are auto-handled, and which need manual follow-up. No silent
caps (see output-discipline / human-oversight).

## Produce the proposal

```
## Codemod — <old> → <new>

**Headline.** <N files would change; M manual cases.> Review the dry-run before applying.

### Files
- codemod: `migrations/<name>.codemod.ts`  ·  runner: `migrate.js` (dry-run default)

### Auto-handled
<the patterns the transform covers>

### Manual follow-up (not auto-transformed)
- `src/.../X.tsx:NN` — dynamic usage; change by hand

### How to apply (human steps)
1. Review the generated files.
2. `node migrate.js` (dry-run) — read the diff.
3. Commit a clean tree, then `node migrate.js --apply`.
4. Run lint + tests.

### Scope
- Generated: transform + dry-run runner. Nothing run, nothing written here.
- Assumed: <static usage; dynamic cases listed as manual>
```

## Closing note

The dry-run is not optional ceremony — it's where you catch the manual cases the transform
can't see. Never wire this into CI to auto-apply; that's exactly what the human-oversight
framework forbids.

## Quality checks

- [ ] Runner defaults to dry-run; `--apply` is explicit and gated.
- [ ] Review banner present; no execution from the skill.
- [ ] Manual/unsafe cases listed, not silently skipped.
- [ ] Transform is idempotent and logs change sites.
- [ ] Output-discipline self-check passed.
